# HARD-2003：启用 UFW 防火墙

UFW 可以采用默认拒绝入站策略，只开放明确需要的服务，但漏掉一个端口就可能导致 SSH、网站、面板、邮件、代理或容器不可用。

执行前应记录当前 SSH 端口、全部监听地址、TCP/UDP 协议、Docker 发布端口以及云厂商安全组。必须先添加 SSH allow 规则，再启用默认拒绝；不能假定 SSH 总在 22 端口。

启用后保持当前连接，并从第二终端重新登录，同时逐一验证网站、控制面板、VPN/代理和容器入口。云安全组与 UFW 是两层规则，两边都必须允许。

失联后从 VPS 控制台运行 `ufw disable` 可先恢复主机侧访问，再根据备份重建规则。
