# HARD-2005：启用 Fail2ban SSH 防护

Fail2ban 根据失败日志临时封禁来源 IP。它适合仍允许密码类认证的公网 SSH，但不能替代密钥登录和防火墙。

动态公网 IP、多人共用出口或反向代理日志识别错误时，管理员也可能被封禁。配置前应确认 SSH 日志后端、真实 jail 名称、当前管理来源 IP，并谨慎设置 `ignoreip`；不要把整个互联网或过大的网段加入白名单。

启动后检查 `fail2ban-client status sshd`，并从第二连接验证正常登录。若被误封，可从仍存活的会话或控制台执行 `fail2ban-client set sshd unbanip <IP>`，再修正规则。
