# VPS Guard Audit 加固说明

VPS Guard Audit 将 Ubuntu/Debian 常见 VPS 加固分成两组：

- **常规安全加固**：通常不会影响当前 SSH 连接，但仍会先备份、验证并复检。
- **连接敏感加固**：可能影响 SSH、网络或现有服务，只能逐项确认，不能“一键全选”。

当前 v6 开发阶段只生成中文计划，不会修改系统。自动处理将在备份、配置验证、延时回滚和复检机制通过测试后开放。

## 连接敏感项目

| 编号 | 项目 | 最主要风险 |
|---|---|---|
| HARD-2001 | 禁止 root SSH 登录 | 没有可用 sudo 用户时失联 |
| HARD-2002 | 禁止 SSH 密码登录 | 密钥不能登录时失联 |
| HARD-2003 | 启用 UFW | SSH 或业务端口未放行 |
| HARD-2004 | 收紧防火墙规则 | 网站、面板、代理或容器端口被阻断 |
| HARD-2005 | 启用 Fail2ban | 管理员来源 IP 被封禁 |
| HARD-2006 | 关闭 SSH 端口转发 | 隧道、代理或开发工具中断 |
| HARD-2007 | 调整 IP 转发或 IPv6 | Docker、VPN、代理、软路由异常 |
| HARD-2008 | 停用服务 | 误停正在使用的业务 |

开始任何连接敏感操作前，请先阅读[连接敏感加固与失联恢复](connection-sensitive.md)。
