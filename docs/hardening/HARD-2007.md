# HARD-2007：调整 IP 转发或 IPv6

这不是普通 VPS 的通用“一键安全项”。Docker、WireGuard、OpenVPN、代理网关、软路由、Kubernetes 和部分云网络都可能依赖 IPv4/IPv6 转发。

工具必须先识别容器、VPN、代理、路由表、nftables/iptables 和 IPv6 地址；无法证明某参数不被依赖时，只给出说明，不自动修改。

变更应写入独立 sysctl 文件，先记录当前值，再使用 `sysctl --system` 应用并验证容器网络、VPN、DNS、IPv4 和 IPv6。网络异常时从 VPS 控制台恢复文件和原始值；不要用重启代替验证。
