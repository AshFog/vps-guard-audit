# HARD-2007：调整 IP 转发或 IPv6

这不是普通 VPS 的通用“一键安全项”。Docker、WireGuard、OpenVPN、代理网关、软路由、Kubernetes 和部分云网络都可能依赖 IPv4/IPv6 转发。

工具必须先识别容器、VPN、代理、路由表、nftables/iptables 和 IPv6 地址；无法证明某参数不被依赖时，只给出说明，不自动修改。

自动动作一次只允许选择一个策略：关闭 IPv4 转发、关闭 IPv6 转发或关闭 IPv6。它要求输入 `NO ROUTING REQUIRED`，写入独立的 `91-vpsga-network-policy.conf`，并同时记录持久化文件与运行时旧值。当前 SSH 使用 IPv6 时会拒绝关闭 IPv6。

应用后必须从第二 SSH 终端确认。超时会恢复配置和原 sysctl 运行时值；若期间又有人改变同一运行时参数，工具会拒绝覆盖新变更并要求从 VPS 控制台检查。
