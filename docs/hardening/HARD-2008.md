# HARD-2008：停用确认不需要的服务

监听端口或服务名称只能说明“它正在运行”，不能证明“它没有用途”。Avahi、CUPS 等服务在普通公网 VPS 上通常不需要，但家庭服务器、远程打印或局域网发现环境可能依赖它们。

工具只处理审计能够明确关联的 `cups` 或 `avahi` 候选组，不接受任意服务名，也不会卸载软件包。用户必须先查看用途清单并输入 `SERVICE NOT NEEDED`；动作会记录每个 service、socket 和 path 单元原来的运行及启动状态，然后逐项停止并禁用原本已启用的单元。

停止后仍需第二 SSH 终端确认。发生超时或验证失败时会恢复各单元原来的运行与启动状态；若等待期间服务状态被其他管理员改变，则不会覆盖该新状态。
