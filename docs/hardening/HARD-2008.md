# HARD-2008：停用确认不需要的服务

监听端口或服务名称只能说明“它正在运行”，不能证明“它没有用途”。Avahi、CUPS 等服务在普通公网 VPS 上通常不需要，但家庭服务器、远程打印或局域网发现环境可能依赖它们。

工具只列出候选服务、监听地址、端口、进程、软件包和依赖关系，不自动全选。用户应确认服务用途，并先 `systemctl stop` 验证业务，再决定是否 `disable`；不应直接卸载软件包。

停止后复查 SSH、网站、数据库、面板、容器、代理和监控。发生业务中断时运行 `systemctl start <unit>`，确认恢复后再决定是否保留开机启动。
