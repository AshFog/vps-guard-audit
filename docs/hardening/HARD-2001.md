# HARD-2001：禁止 root SSH 登录

这项设置把 `PermitRootLogin` 改为 `no`，减少直接针对最高权限账户的攻击面。

**不要执行的情况：** 你目前只使用 root 登录；没有普通管理用户；sudo 尚未验证；无法使用 VPS 网页控制台。

执行前应创建普通用户、加入 sudo 组、复制并修正其 `authorized_keys`。工具会记录当前实际使用的 SSH 地址和端口；修改后，必须从第二个终端通过同一入口登录并运行带 `sudo` 的确认命令。

修改目标为独立的 SSH drop-in 文件。写入后先运行 `sshd -t`，再 reload SSH；当前连接保持不动。只有第二个终端以普通用户成功登录并使用 sudo 后，才能确认完成。

若验证失败，应在自动回滚倒计时结束前选择立即恢复；如果已经失联，通过 VPS 控制台删除该 drop-in 文件，运行 `sshd -t` 后重新加载 SSH。
