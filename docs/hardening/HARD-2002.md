# HARD-2002：禁止 SSH 密码登录

这项设置关闭 `PasswordAuthentication` 和 `KbdInteractiveAuthentication`，阻止公网密码猜测。

**不要执行的情况：** 没有公钥；私钥只保存在一台不可靠设备；密钥带来的新连接尚未实际验证；云厂商初始化或运维系统依赖密码认证。

工具必须找到至少一个有效的 `authorized_keys`，检查文件所有者与权限，并要求用户保留当前窗口，从第二个终端完成一次全新的登录。建议明确禁止客户端回退到密码：`ssh -o PasswordAuthentication=no -o KbdInteractiveAuthentication=no 用户@服务器 -p 端口`。仅仅“文件存在”不代表密钥可用。

写入独立 SSH drop-in 后先运行 `sshd -t`，再 reload。若第二连接失败，不要退出当前窗口；让延时任务自动恢复，或立即执行回滚。
