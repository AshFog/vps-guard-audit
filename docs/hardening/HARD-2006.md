# HARD-2006：关闭 SSH 端口转发

`AllowTcpForwarding no` 可以减少被入侵账户用作隧道的能力，但会直接影响 SSH 隧道、远程开发、数据库端口映射、代理转发和部分运维工具。

网站服务器不等于“不使用转发”。执行前检查当前团队和自动化是否使用 `ssh -L`、`ssh -R`、`ssh -D`、VS Code Remote、数据库隧道或跳板机。

如果只需限制部分用户，应优先在 `Match User` 或 `Match Group` 中精确设置，而不是全局关闭。

自动动作要求先运行用途检查并输入 `NO SSH FORWARDING`，然后在受管理 drop-in 中设置 `AllowTcpForwarding no`。写入后执行 `sshd -t` 与 `sshd -T`，reload 失败立即恢复；成功后仍需第二 SSH 终端确认。
